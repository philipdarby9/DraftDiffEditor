# Audit Report - Draft Diff Editor / 2026-06-07 / 50febac

## Executive Summary

Overall risk: Medium-High for a private local app, driven by local-server exposure, non-atomic multi-file saves, and large-document diff/render cost.
Highest-value fix: make persistence atomic and explicitly bind/guard the write-capable local API.
Counts: 0 Critical, 4 High, 10 Medium, 2 Low.
Scope: `server.js`, `public/app.js`, `public/panel.js`, Electron shell, scripts, packaging config, dependency metadata, and repo-local skill files.
Key limitation: no browser/Electron runtime profiling or packaged build was run; findings are static plus syntax/dependency checks.

## Remediation Status

Applied after this audit:

- Fixed the standalone local-server boundary: `npm start` now binds to `127.0.0.1` by default and write-capable API requests require loopback Host/Origin unless `DRAFT_DIFF_ALLOW_REMOTE=1` is set.
- Fixed static path containment by replacing prefix matching with `path.relative` containment.
- Moved core project, export, linked-text, text-file-state, and version-history sidecar writes onto the existing atomic-write path, with a Windows OneDrive fallback for paths that can leave undeletable placeholder temp files.
- Added a positive version-history sidecar path cache to avoid repeated folder scans/parses on the common save/read path.
- Moved full version-history summary generation to a worker thread with progress/error propagation.
- Routed common compare-panel refresh paths through the progressive renderer.
- Added browser-side diff token/block caching, bounded pair-result caching, faster block-term matching, and guardrails for block/token comparison cells. Very large comparisons now show an explicit skipped-detail state rather than freezing the renderer.
